当前项目瘦身复核结论

更新时间: 2026-04-23
目标: 在保证功能可用和易迭代的前提下，保留个人使用的管理后台能力，同时支持外部 API 接入；不引入多 server、微服务、分布式复杂度。

结论摘要

- 这份文档替代旧版结论。旧版中关于“无权限声明即放行”、DictItem/SystemConfig/ApiApp 缺少权限声明、AuthModule 重复全局 JWT Guard 等问题已经陈旧。
- 当前权限模型已经收紧：非公开接口必须显式声明 @Public、@AllowAuthenticated、@RequirePermissions 或 @RequireApiScopes。
- P1/P2 功能风险已修复；本轮已继续移除偏脚手架/平台化的模块能力，让项目更接近个人单体后台 + 外部 API 的目标。

已陈旧的旧结论

- “无权限声明即放行”已经修复。PermissionsGuard 现在默认拒绝未声明权限的非公开接口。
- DictItemController 已补齐 @RequirePermissions。
- SystemConfigController 已补齐 @RequirePermissions。
- ApiAppController 已补齐 @RequirePermissions。
- AuthModule 中重复注册全局 JwtAuthGuard 的问题已处理。
- Redis、EventEmitter、分布式锁主链路已经移除；当前 src 下未发现 Redis/EventEmitter 使用。
- Notification 渠道已收敛为 Bark 和飞书。
- 文件存储策略已收敛为 local 和 oss。

已修复 P1 Bug / 功能风险

1. OpenAPI 路由版本可能重复
   - 位置: src/main.ts:36，src/modules/open-api/controllers/open-api.controller.ts:21
   - 修复: OpenApiController 改为 open，由全局 URI versioning 统一生成 /api/v1/open。

2. 全局响应包装可能破坏文件下载
   - 位置: src/core/interceptors/transform.interceptor.ts:21，src/modules/file/controllers/file.controller.ts:272
   - 修复: TransformInterceptor 跳过 StreamableFile、Buffer、Node Readable stream。

3. 公开注册仍存在
   - 位置: src/modules/auth/controllers/auth.controller.ts:41
   - 修复: 删除 AuthController 的公开 register 路由，用户创建复用 UserController 的 user:create 权限接口。

4. 首次初始化没有菜单数据
   - 位置: src/modules/user/services/admin-bootstrap.service.ts:25，src/migrations/1730000000000-InitSchema.ts
   - 修复: AdminBootstrapService 在空库首次启动时创建基础菜单，避免首次登录菜单为空。

已修复 P2 Bug / 隐患

1. LoggerService context 是单例可变字段
   - 位置: src/shared/logger/logger.service.ts:82
   - 修复: setContext 不再写入共享实例字段，避免跨 service 污染日志 context。

2. 用户权限缓存仍有两套 key
   - 位置: src/core/guards/permissions.guard.ts:141，src/modules/user/services/user.service.ts:459
   - 修复: UserService 与 PermissionsGuard 统一使用 CACHE_KEYS.USER_PERMISSIONS(userId)。

3. 数据库配置 extra 被覆盖
   - 位置: src/shared/database/database.module.ts:17，src/config/configuration.ts:43
   - 修复: DatabaseModule 通过 buildDataSourceOptions 合并并保留 getDatabaseConfig 的 extra。

4. readyz 不会按注释返回 503
   - 位置: src/modules/health/controllers/health.controller.ts:51
   - 修复: 依赖 unhealthy 时抛 ServiceUnavailableException。

5. API 调用日志统计不可信
   - 位置: src/modules/api-auth/strategies/simple-api-key.strategy.ts:107
   - 修复: API Key 策略不再记录固定 200/0ms 的虚假调用日志；后续瘦身中已删除调用日志统计和复杂限流，仅保留 API Key 认证。

6. Excel 类型残留
   - 位置: src/config/configuration.ts:169，src/config/config.validation.ts:185，src/modules/file/services/file.service.ts:112，src/common/utils/file.util.ts:202
   - 修复: 默认 allowedTypes、配置校验、FileService 默认值和 FileUtil 文档分类均移除 .xls/.xlsx。

7. local 存储下签名 URL 接口必然失败
   - 位置: src/modules/file/storage/local-storage.strategy.ts:109，src/modules/file/controllers/file.controller.ts:217
   - 修复: 先改为 local 文件返回受鉴权保护的 /api/v1/files/:id/download；后续瘦身中已直接删除 signed-url 接口，统一走 download 鉴权下载。

剩余注意点

1. Notification 仍有数据库通知、外部渠道发送、离线发送配置。当前已经只保留飞书和 Bark，是否继续瘦身取决于个人后台是否真的需要站内通知。
2. Menu 仍保留数据库表和管理接口。当前选择是“极简菜单表”，如果前端路由完全静态，后续还能删除 Menu 模块和 role_menus。
3. Permission CRUD 仍保留。当前选择是“显式权限声明 + 数据库角色授权”，如果个人使用长期只需要 super_admin，可进一步删除权限管理 UI，仅保留代码常量。
